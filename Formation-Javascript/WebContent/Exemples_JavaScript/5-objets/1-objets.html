POO :



Pour comprendre les mécanismes basiques de la POO
essayons de visualiser les objets du monde qui nous entoure
sous un angle bien spécifique :
• à un instant t, un objet du monde réel est dans un certain état.
• cet état varie en fonction des interactions de l'objet dans le monde réel.

• En pratique designer une application consiste à identifier :
• les objets existants dans le contexte de l'application.
• ce qui définit l'état de ces objets.
• les interactions entre les objets de l'application.


    Exemple Une bicyclette a :

    
• des "state" (états): vitesse courante, plateau, pignon, ... 

• des "behaviour" (comportements): freiner, changer de plateau, changer de pignon, ...
